Segment Control

Usage

Required

The "segment.js" file is not required per se. It's just a controller for btn-group
Every JS controller can be used for every btn-group! 

            <head>
  ...
  <link rel="stylesheet" href="segment.css" />
  <script type="text/javascript" src="segment.js"></script>
  ...
</head>

HTML Layout

The attribute "data-group-relationship" controls the relationship between the btns. Default is "relative"

Relative relationship

            ...
  <div class="btn-group" data-group-relationship="relative">
    <button class="btn btn-segment active">Segment A</button>
    <button class="btn btn-segment">Segment B</button>
    <button class="btn btn-segment">Segment C</button>
  </div>

  <br /><br />

  <div class="btn-group" data-group-relationship="relative">
    <button class="btn btn-default active">Btn A</button>
    <button class="btn btn-default">Btn B</button>
    <button class="btn btn-default">Btn C</button>
  </div>
...

JavaScript Reference

"segment.js" is just a global JS controller for every btn-group. Therefore you can use it not only for segments.

Rules

  1. A "btn-group" element has the prototype function "updateBtnGroup()"
  2. A "btn-group" element has the event "activate"

            var myBtnGroup = document.getElementsByClassName("btn-group")[0]; //Example for an "btn-group"
var myBtn = myBtnGroup.childNodes[0]; //Example for an "btn" within our "btn-group"

//Update the btn-group. First argument is the "btn" element. Use it for manual updating a btn-group
myBtnGroup.updateBtnGroup(myBtn);

//Event Listening for changes
myBtnGroup.addEventListener("activate", function() {
  event.detail;                   //Event details
  event.detail.targetBtn;         //The clicked "btn" element
  event.detail.groupRelationship; //Relationship between the "btn" elements
  console.log(event.detail);
}, false);
